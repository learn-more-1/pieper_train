# 模型架构对比

## 一眼看懂区别

### 🔴 原模型 (PieperCausalIK)

```
输入: 历史[10帧] ────────────┐
                             ├──► [隐式耦合] ──► 角度
输入: 目标位姿[7] ───────────┘

问题: 历史为零 → 模型冻结 → 输出零 → 机器人不动
     (耦合关系被历史淹没)
```

### 🟢 新模型 (ExplicitCouplingIK)

```
输入: 目标位姿[7] ──► 运动意图 ──► [显式耦合图] ──► 角度

优势: 无需历史！耦合显式编码在图结构中！
     (耦合关系由物理结构保证)
```

## 核心区别图解

### 原模型：耦合隐含在历史中

```
┌─────────────────────────────────────────────────────────┐
│  历史 [t-9] [t-8] ... [t-1] [t]                         │
│    ↓     ↓         ↓     ↓                              │
│  ┌──────────────────────────────────────┐              │
│  │    时序编码器 (LSTM/GRU/Conv1D)       │              │
│  │    学出来的耦合关系 (看不见)           │              │
│  └────────────────┬─────────────────────┘              │
│                   ↓                                     │
│              当前状态估计                               │
│                   │                                     │
│                   ↼────────── 目标位姿 (弱影响)         │
│                   ↓                                     │
│              预测角度                                   │
└─────────────────────────────────────────────────────────┘

问题: 历史为零时，当前状态估计也为零
      目标位姿影响太弱
      输出 ≈ 零
```

### 新模型：耦合显式在图结构中

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  目标位姿 [x,y,z,qx,qy,qz,qw]                           │
│       │                                                 │
│       ▼                                                 │
│  ┌─────────────────────┐                                │
│  │   运动意图编码器     │                                │
│  │   "要做什么运动"     │                                │
│  └──────────┬──────────┘                                │
│             │                                           │
│             ▼                                           │
│  ┌─────────────────────────────────────────────────┐   │
│  │              显式耦合图                          │   │
│  │                                                 │   │
│  │   J0──J1──J2──J3──J4──J5──J6                  │   │
│  │   │        │   │       │                      │   │
│  │   └────────┘   └───────┘                      │   │
│  │   功能耦合边 (预定义的物理关系)                 │   │
│  │                                                 │   │
│  │   • 肩影响肘 (Pieper)                          │   │
│  │   • 肘影响腕 (Pieper)                          │   │
│  │   • 运动链 (物理连接)                          │   │
│  │                                                 │   │
│  └──────────────────┬──────────────────────────────┘   │
│                     │                                   │
│                     ▼                                   │
│  ┌─────────────────────────────────────────────────┐   │
│  │           耦合约束解码器                         │   │
│  │                                                 │   │
│  │   Shoulder [3] ──g──► Elbow [1]               │   │
│  │                          │                      │   │
│  │                          g──► Forearm [1]      │   │
│  │                                 │               │   │
│  │                                 g──► Wrist [2] │   │
│  │                                                 │   │
│  │   g = 门控 (可学习的耦合强度 0~1)               │   │
│  │                                                 │   │
│  └──────────────────┬──────────────────────────────┘   │
│                     │                                   │
│                     ▼                                   │
│              预测角度 [7]                               │
│                                                         │
│   + 耦合信息:                                           │
│     • elbow_shoulder_coupling: 0.52                    │
│     • forearm_elbow_coupling: 0.48                     │
│     • wrist_forearm_coupling: 0.51                     │
│                                                         │
└─────────────────────────────────────────────────────────┘

优势: 无需历史！耦合由图结构显式保证！
      目标位姿直接驱动！
      可解释的耦合强度！
```

## 数据流对比

### 原模型

```
历史 [10,7] ───────────────────────┐
                                   ├──► 预测 [7]
目标位姿 [7] ─── (弱影响) ────────┘

零历史输入 ──► 预测 ≈ 零 ❌
```

### 新模型

```
目标位姿 [7] ─── (强驱动) ───► 预测 [7]

零历史输入 ──► 预测正常 ✅
```

---

## 训练对比

| 项目 | 原模型 | 新模型 |
|------|--------|--------|
| **输入** | 历史+位姿 | **纯位姿** |
| **数据需求** | 时间序列 | **位姿-角度对** |
| **耦合来源** | 从历史学习 | **显式编码** |
| **可解释性** | 黑盒 | **白盒** |
| **历史依赖** | 强 (必须) | **无** |

---

## 一句话总结

> **原模型**: 从历史轨迹中"悟出"关节如何配合
> **新模型**: 把已知的物理配合关系"告诉"模型

就像：
- 原模型：让学生从历史试卷中总结物理公式
- 新模型：直接把物理公式写在黑板上，教学生怎么用
