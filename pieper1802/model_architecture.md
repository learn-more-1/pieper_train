## 核心创新点

### 1. 运动意图编码
```
目标位姿 ──► 深层MLP ──► 运动意图向量

不是直接映射到位姿，而是编码"要做什么运动"
```

### 2. 显式耦合图
```
传统方法: 耦合隐含在历史数据中
本方法:   耦合显式编码在图结构中

边类型:
- 运动链边: 物理连接 (J0-J1-J2-J3-J4-J5-J6)
- 功能耦合: Pieper准则 (J0/J1→J3, J2→J5, J3→J5)
```

### 3. 耦合约束解码
```
关节分组 + 门控融合

Shoulder ──g──► Elbow ──g──► Forearm ──g──► Wrist
               
g = Sigmoid(MLP([父组特征, 子组特征]))

可解释的耦合强度: 0~1 之间
```

## 与原模型对比

| 特性 | 原模型 (PieperCausalIK) | 显式耦合模型 |
|------|------------------------|-------------|
| 输入 | 历史帧[10,7] + 位姿 | **纯位姿[7]** |
| 耦合方式 | 隐式 (GNN学习) | **显式 (图结构)** |
| 历史依赖 | 强 (必须) | **无 (可选)** |
| 可解释性 | 黑盒 | **白盒 (可见耦合强度)** |
| 训练数据 | 需要时间序列 | **只需要位姿-角度对** |

## 数据流

```
Forward Pass:
1. 目标位姿 ──► 意图编码器 ──► 意图向量[256]
2. 意图向量 ──► 分发到7关节 ──► 关节特征[7, 128]
3. 关节特征 ──► 耦合图消息传递 ──► 耦合特征[7, 128]
4. 耦合特征 ──► 分组解码 ──► 关节角度[7]

输出:
- pred_angles[7]: 预测的关节角度
- coupling_info: 各组耦合强度 (可解释)
```

## 训练目标

```
总损失 = IK损失 + FK损失 + 耦合正则化

IK损失: 预测角度 vs 真实角度 (MSE)
FK损失: FK(预测) vs 目标位置 (MSE)
耦合正则化: 鼓励耦合强度在0.5左右 (不过强也不过弱)
```

## 推理时的优势

```
传统模型:
历史为零 ──► 预测接近零 ──► 机器人不动

本模型:
目标位姿 ──► 生成运动意图 ──► 显式耦合求解 ──► 合理角度
     
无需历史！耦合关系由图结构保证，不是从历史学习！
```
